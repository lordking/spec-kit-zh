# Spec Kit

*更快地构建高质量软件。*

**通过规格驱动开发（Spec-Driven Development），让组织专注于产品场景，而不是编写重复性代码。**

## 什么是规格驱动开发？

规格驱动开发**颠覆**传统的软件开发模式。几十年来，代码一直是王道——规格只是我们在开始"真正工作"（编码）之前搭建并丢弃的脚手架。规格驱动开发改变了这一点：**规格变得可执行**，直接生成可工作的实现，而不仅仅是指导实现。

## 核心思想

规格驱动开发是一个结构化的流程，强调：

- **意图驱动开发**：在"如何做"之前先通过规格定义"*什么*"
- **丰富的规格创建**：使用护栏和组织原则
- **多步细化**：而不是一次性从提示生成代码
- **重度依赖**：高级AI模型的规格解释能力

## 开发阶段

| 阶段 | 重点 | 关键活动 |
|-------|-------|----------------|
| **0到1开发**（"绿地开发"） | 从零生成 | <ul><li>从高级需求开始</li><li>生成规格</li><li>规划实施步骤</li><li>构建生产就绪的应用</li></ul> |
| **创造性探索** | 并行实现 | <ul><li>探索多样化解决方案</li><li>支持多种技术栈和架构</li><li>尝试UX模式</li></ul> |
| **迭代增强**（"棕地开发"） | 棕地现代化 | <ul><li>迭代添加功能</li><li>现代化遗留系统</li><li>调整流程</li></ul> |

## 实验目标

我们的研究和实验重点关注：

### 技术独立性

- 使用多样化的技术栈创建应用
- 验证规格驱动开发是一个不依赖于特定技术、编程语言或框架的过程

### 企业约束

- 演示关键任务应用开发
- 纳入组织约束（云提供商、技术栈、工程实践）
- 支持企业设计系统和合规要求

### 以用户为中心的开发

- 为不同的用户群体和偏好构建应用
- 支持各种开发方法（从 vibe-coding 到 AI原生开发）

### 创造性和迭代流程

- 验证并行实现探索的概念
- 提供强大的迭代功能开发工作流
- 扩展流程以处理升级和现代化任务
